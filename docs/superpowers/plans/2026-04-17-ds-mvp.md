# Design System MVP — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` (or `superpowers:subagent-driven-development`) to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `@mande/ui` the single source of truth for every component rendered in the playground, then extend the DS to React Native via a shared-tokens package.

**Contract:**
- Playground imports only from `@mande/ui`. Zero hand-rolled duplicates.
- DS is the single source of truth. Any visual/behavioral tweak lives in `packages/ui`; Storybook and playground render the same component.
- If the DS is missing a prop/variant the playground needs, add it to the DS component — never fork.
- Definition-of-done for any playground tweak includes promoting the change back into `packages/ui` before merge.

**MVP surface area:** three active screens (`home`, `chat`, `career-discovery`) plus a revised playground index that separates Active from Inactive screens. Onboarding and Settings are parked as Inactive.

**Targets:** responsive web first (Phase 1), then React Native via NativeWind + shared tokens (Phase 2).

**Tech stack:** Next.js 15 App Router, Tailwind v4, Radix UI + shadcn conventions, Central Icons via `<Icon>`, `motion` v12 / `tw-animate-css`. Phase 2 adds NativeWind + RN + Expo.

## Phase 2 — React Native track (NativeWind + shared tokens)

> Do not start Phase 2 until Phase 1 is complete and merged. RN on top of an inconsistent web DS multiplies the mess.

**Approach:** Option (c) — NativeWind + shared tokens layer.

- Web DS (`@mande/ui`) stays as-is: Tailwind v4, Radix, shadcn.
- New `packages/tokens` exports pure TypeScript tokens (colors, spacing, typography, motion, radii). Consumed by both web and native DSes.
- New `packages/ui-native` mirrors component names and prop APIs with React Native + NativeWind implementations. Same Tailwind class vocabulary.
- Native interactions (sheet vs drawer, haptics, scroll behaviors, keyboard avoidance) live in `@mande/ui-native` as native-first implementations, not web-translated.

### Task 9: Extract `packages/tokens`

- [ ] Create `packages/tokens` as a new workspace package.
- [ ] Export: color palettes (primary/blue/green/blush/orange/neutral across 50–900 shades), radii (rounded-1/2/3), spacing scale, typography scale, motion (durations, easings, springs).
- [ ] Convert literals in `packages/ui/src/tokens/motion.ts` and any Tailwind config into token-consuming form.

### Task 10: Refactor web DS to consume tokens

- [ ] Update `packages/ui` to import from `@mande/tokens`.
- [ ] Verify no visual regression in Storybook.

### Task 11: Scaffold `packages/ui-native`

- [ ] New workspace package with React Native + NativeWind configured.
- [ ] Mirror DS exports: `Button`, `Input`, `Select`, `Sidebar`, `Avatar`, `Card`, `Badge`, `Chip`, `Icon`, `Progress`.
- [ ] Consume `@mande/tokens` for all values.
- [ ] Document parity table: component name, web status, native status, known divergences.

### Task 12: Expo demo app

- [ ] Optional: `apps/native-playground` (Expo) that renders the Active screens (chat, career-discovery, home) using `@mande/ui-native`.
- [ ] Surfaces gaps in native parity and validates the shared-tokens approach end-to-end.

### Phase 2 — done when:

- [ ] `packages/tokens` is the source of truth for design values; web and native both consume it.
- [ ] Every component shipped in Phase 1 has a native counterpart with matching prop API.
- [ ] An Expo demo renders a recognizable parallel of the web playground.
